# Privacy Model

## 隐私原则

候选人之间完全互相不可见。候选人不知道某个时间不可选的真实原因，只看到不可选状态。

## 候选人可见

- 自己的姓名、邮箱。
- 面试组名称和公开说明。
- 自己当前有效提交的可用时间。
- 自己填写的候选人备注。
- 自己的修改审核状态。
- 已安排面试的候选人可见说明、会议地点或链接。

## 候选人不可见

- 其他候选人。
- 其他候选人是否选择某时间。
- 某时间不可选是否因为关闭、锁定、预约或内部原因。
- 管理员私有备注。
- 管理员内部备注。
- 审计日志。

## 管理员私有备注

`CandidateAdminNote` 只能通过管理员 API 读取和写入。候选人页面、候选人 API、候选人响应 payload 中绝对不能包含该表内容。

## 测试要求

必须有自动测试序列化候选人 DTO，并断言响应中没有：

- `adminNotes`
- `internalNote`
- `reasonInternal`
- `availableCandidateCount`
