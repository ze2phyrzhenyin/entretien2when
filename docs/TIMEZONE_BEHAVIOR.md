# 时间与时区行为

本项目的时间处理规则：

- 数据库中的 `startAt`、`endAt`、提交时间、审计时间等均按 UTC 存储。
- 面试组的 `timezone` 是该组生成时间段和默认展示的业务时区。
- 批量生成时间段时，管理员输入的日期和时间会按面试组时区解释，再转换成 UTC 写入数据库。
- 候选人端和管理员端的时间展示会从 UTC 自动换算到当前显示时区。
- 显示时区可切换：
  - 面试组时区；
  - 浏览器时区；
  - 手动选择常用 IANA 时区。
- 时区选择保存在浏览器本地，只影响当前用户看到的时间，不改变数据库、预约锁定、审核或候选人提交结果。

## 隐私边界

- 候选人端只显示自己的可选时间、已提交时间和自己的预约时间。
- 切换时区不会暴露其他候选人、锁定原因、管理员备注或内部备注。

## 验收重点

- 同一个 UTC 时间在 `Asia/Shanghai` 和 `Europe/Paris` 下应显示不同本地时间。
- 切换时区后，候选人时间格、已提交时间、预约时间、管理员时间段、预约列表、审核对比和操作日志应同步更新。
- 创建和编辑面试组的时区必须是有效 IANA 时区，例如 `Asia/Shanghai`、`Europe/Paris`。
